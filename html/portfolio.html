Something2Watch

Idea

I came up with this idea through personal necessity. I love watching tv, but so often its tough to find something good to watch. There are lots of similar sites out there, but they all seemed overly complicated; asking specifics about genere or certificate (amongst a myriad of other things). My idea is nice and simple; what is something you've enjoyed before? Here are some similar shows to choose from!

Designs

I had a good idea in my mind of how I wanted the site to look. I design and make all sites that I work on using the 'mobile first' methodology; but this site more than most, I knew would get of traffic from mobile and tablet (most people sat on their sofa wondering what to watch tend to have their phone or tablet to hand!). I fleshed out some wireframes and user flow (small, medium and large) in Balsamiq and decided on the font and colour with inspiration from a recent copy of Web Designer magazine.

Challenges

The biggest challenge was finding the right API to get the data. In the end, I wasn't able to find a single data source that gave me everything I wanted; themoviedb came close; it provided a comprehensive query endpoint to find suggestions using any number of different metrics, but the one downside was that it used it's own rating system, rather than the more farmiliar (and much more widely used) IMDB and Rotten Tomatoes, so I ended up using omdbapi for the basic search and most of the UI data (as well as that important sort by rating), but fell back on themoviedb for the impressive search api.

The second challege came with getting the data out of the api's. Because of the need to switch between the two api's to retrieve and display the results, there are 5 asyncronus api calls, that all depend on the previous in order to work. Initially i tried doing this in the redux container; checking that the previous API had returned it's result using componentWillReceiveProps and firing off the next call when it had; but that got very messy very fast; so instead I decided on chaining promises in the action creator instead. I felt this was the best all round solution as it keeps the container nice and concise whilst still not having an overly complicated action.

Further Development Ideas

This one is easy, if I can find an API that provides the data, I would love to be able to show which platforms each result is available on; whether it be Netflix, Amazon Prime, iPlayer, etc...

IfOnly

Idea

I came up with the idea for this site when chatting to a friend; "I wonder, if someone had invested some money in Apple for me on the day I was born, what would it be worth now?". It turns out, it would be worth about 150 times more today than it was then.

Designs

I think I spent more time thinking about how to make this idea fun for the end user, rather than the actual design of it; and honestly? I think that shows. I am very pleased with the story telling aspect of it though.

Challenges

The hardest part of this was the time spend on getting the inputs to work how I wanted them to. They grow and shrink to fit the text that the user adds; they are responsive across different viewports; utilising mobile and tablet devices native datetime pickers, and a jQuery plugin to achieve the same thing on desktop.

I also took care when it came to loading the background image. There are 5 potential images that can be used, the site randomises which one is displayed each time the page is loaded. The image is then loaded asyncronously and fades in when it has finished downloading. It's turns out to be a fairly simple bit of javascript, but it took a decent amount of thought!

Further Development Ideas

I've already done a bit of further development; I added the bottom section, calculation when would have been the best possible day for me to ahve cashed out! I also added the UK FTSE100 stock exchange, to go wiht the existing US S&P500. I do still have a couple of ideas though; I would definitely like to redesign the UI to use more material design aspects(I've never been totlly happy with it, especially the loading spinner!). I would also like to add CryptoCUrrencies to the list of available investments.

StudentBeans

Studentbeans is a website and app that provides students with discounts to thousands of different brands all over the world. I mostly worked on the Single Sign On and user management areas of the site. I worked closely with a Ruby on Rails back end developer and UX and Design specialist to impliment changed required as a result of the GDPR legislation, and general improvements to the signup and verification process; where success was measured by a decrease in the percentage drop off of user signups to user verifications.

Coding standards were taking very seriously at StudentBeans, merge requests and code reviews were are large part of the job. Having your own code checked and critiqued as well as reviewing and critiquing other developers code made me a better developer. On top of this, each squad was provided with a problem to solve, rather than simply being given a brief to fulfill, this bread teamwork and collaberation between all members of the squad.

Jaguar & Land Rover Configurator

The configurator was an amazingly comlext project to work on. It involved vast amounts of data much of which would change after a couple of site interactions by the user. The raw data was manipulated by a series of custom nodeJS middlewares and stored in a Redux state. Another challenge of the project was getting the sheer amount of data to fit in the available screen real-estate; especially on mobile. Many hours were spent wiht the designer making small tweaks to maximise the space utilised.

The main aspect I worked on was the new 'Reccomended Choice' initiative, which was largely driven direcly by the client. This entaled each market being able to set a 'reccomended choice' for each category within the configuration. Each one would be identified to the user. An option was also added to view a the fully configured 'reccomended choice' vehicle, without having to do any configuration at all.

Jaguar & Land Rover DX

This is the main 'Digital Experience' platform for Jaguar and Land Rover, across all markets worldwide. I was involved in many areas of the site, but the main aspect I worked on was a complete refresh of the Retailer Locator component. This required a large degree of attention to detail as it is considered one of the most important areas of the site, as it is the main way to direct customers to individual retailers to begin the sales process.

Wallpapers

Idea

I love having colorful and interesting wallpapers on my mac! So I decided to make a little app that would allow me to download the current trending images in the EarthPorn sub-reddit on Reddit. I then setup my mac to cycle through the images changing my wallpaper every 30 minutes!

Challenges

There wasn't any massivley challenging aspects to this. Probably the biggest headache was working with the notoriously fiddley file system implimentation in nodeJS.

Further Development Ideas

I have already implimented a settings file so that the user can easily configure things such as which subReddit to get the images from, where the downloaded images should be saved and the minimum image dimensions that should be allowed. If I was to develop it further, I would like to investigate a way of automatically setting the downloaded images as the wallpaper, rather than using the built in mac display settings tool; and have the app run periodically automatically, rather than having to manually set it as a scheduled task.
